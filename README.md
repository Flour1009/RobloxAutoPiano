# README для скрипта
# MIDI → Virtual Piano / Roblox Player
Описание скрипта, его функционала и инструкции по использованию
Этот скрипт автоматически проигрывает MIDI-файлы через обычную клавиатуру в виртуальном пианино, например в Roblox Piano. Он читает ноты из MIDI, адаптирует их под доступный диапазон клавиш и эмулирует нажатия в реальном времени.
# Что делает скрипт
Этот скрипт нужен для того, чтобы автоматически воспроизводить музыку из MIDI-файлов в виртуальном пианино, которое управляется обычной клавиатурой. На практике это особенно удобно для Roblox Piano и похожих игр, где каждая нота привязана к конкретной кнопке на клавиатуре. Вместо того чтобы нажимать клавиши вручную, пользователь запускает программу, выбирает MIDI-файл, а затем скрипт сам разбирает композицию и воспроизводит её через эмуляцию нажатий. По сути, он превращает MIDI-дорожку в последовательность реальных клавиатурных событий, которые отправляются в окно игры или другого приложения с виртуальным пианино.
# Как работает скрипт
Работа начинается с загрузки MIDI-файла. Файл можно передать через аргумент командной строки или выбрать вручную через графическое окно. Для чтения используется библиотека mido, но в коде предусмотрена и дополнительная безопасная загрузка: если обычное чтение не срабатывает, скрипт пытается вручную найти MIDI-заголовок MThd внутри файла и прочитать данные заново. Это помогает открывать некоторые нестандартные или не совсем корректно сохранённые MIDI-файлы. После загрузки файл разбирается на события нот: определяется время начала и конца каждой ноты, её номер, сила нажатия и канал. Скрипт учитывает изменения темпа, поэтому воспроизведение по времени остаётся максимально близким к оригиналу. Ударные автоматически пропускаются, потому что стандартный барабанный канал MIDI обычно не подходит для виртуального пианино.
# Режимы работы
Скрипт поддерживает два режима: melody и full. В режиме melody он пытается извлечь только основную мелодическую линию из всего MIDI. Это полезно, когда оригинальный файл слишком насыщенный и содержит аккорды, сопровождение, фоновые партии и дополнительные инструменты. В таком случае скрипт анализирует одновременно звучащие ноты и выбирает одну наиболее подходящую. Можно выбрать стратегию smooth, при которой мелодия получается более плавной и с меньшим количеством резких скачков, либо highest, при которой в каждый момент берётся самая высокая нота. Отдельный параметр stickiness влияет на то, насколько неохотно алгоритм будет переключаться между соседними нотами. В режиме full скрипт не выделяет одну линию, а сохраняет несколько одновременных нот, благодаря чему можно услышать аккорды и часть сопровождения. При этом есть ограничение на максимальное число одновременно зажатых клавиш, чтобы виртуальное пианино не перегружалось.
Преобразование нот в клавиши
Одна из главных задач скрипта — сопоставить MIDI-ноты с клавишами обычной клавиатуры. Для этого используется заранее заданная последовательность белых клавиш: 1234567890qwertyuiopasdfghjklzxcvbnm. Чёрные клавиши формируются через сочетание Shift с той же кнопкой. Благодаря этому создаётся расширенная хроматическая клавиатура, на которую можно перенести MIDI-ноты. Когда скрипт обрабатывает очередную ноту, он определяет, попадает ли она в доступный диапазон, и если да, то выбирает соответствующую клавишу и при необходимости добавляет Shift. Затем это превращается в обычное нажатие и отпускание клавиши через pydirectinput.
# Автоматический подбор диапазона
Так как MIDI-файлы могут содержать слишком высокие или слишком низкие ноты, которые не помещаются в доступную клавиатуру, скрипт умеет автоматически подбирать рабочий диапазон. Он перебирает несколько возможных стартовых позиций и варианты транспонирования, оценивая, какой вариант позволит сохранить больше нот и меньше терять содержимое композиции. Кроме того, учитывается, насколько сильно ноты упираются в края диапазона. В результате скрипт сам подбирает значения base_midi и transpose. Если автоматический результат не устраивает, эти параметры можно задать вручную.
# Эмуляция нажатий и управление
Для воспроизведения скрипт использует библиотеку pydirectinput, которая отправляет в систему нажатия так, как будто пользователь нажимает клавиши вручную. Белые ноты воспроизводятся обычным нажатием кнопки, а для чёрных на короткий момент удерживается Shift. Перед запуском скрипт пытается найти окно Roblox и вывести его на передний план, чтобы ввод ушёл именно в игру. После этого он ждёт горячую клавишу старта. По умолчанию воспроизведение запускается по F8, а останавливается по F9. Когда пользователь нажимает клавишу старта, скрипт начинает воспроизводить все действия строго по времени. Если нажать клавишу остановки, воспроизведение прерывается, а все удерживаемые клавиши принудительно отпускаются.
# Установка и запуск
Для работы нужен Python 3 и библиотеки mido, pydirectinput, pygetwindow и pynput. Также используется tkinter для окна выбора файла. Установить зависимости можно командой ниже. После этого скрипт можно запустить либо без аргументов, чтобы выбрать MIDI-файл вручную, либо сразу передать путь к файлу через --file. Перед запуском воспроизведения желательно открыть Roblox или другую программу с виртуальным пианино, переключить её в оконный режим, включить английскую раскладку клавиатуры и кликнуть по самому пианино, чтобы оно принимало ввод.
# Основные параметры
Скрипт позволяет настраивать почти все важные параметры через аргументы командной строки. Через --start-key и --stop-key задаются клавиши запуска и остановки. Параметр --speed управляет скоростью: значения больше 1 замедляют воспроизведение, а меньше 1 ускоряют. Аргумент --mode переключает режимы melody и full. Параметр --melody-strategy задаёт способ выбора мелодической ноты, а --stickiness регулирует плавность переключений. Через --max-poly можно ограничить число одновременно зажатых нот. Параметры --min-hold и --max-hold задают минимальную и максимальную длительность удержания клавиш. Аргументы --base-midi и --transpose позволяют вручную настроить диапазон, а --channels дают возможность оставить только определённые MIDI-каналы.
# Как пользоваться на практике
Обычный сценарий использования выглядит так: пользователь открывает Roblox Piano, включает английскую раскладку и делает окно активным. Затем запускает скрипт, выбирает MIDI-файл и смотрит служебную информацию в консоли. Скрипт показывает, сколько нот найдено, сколько из них попадает в диапазон, какой base_midi выбран, какое транспонирование применилось и какая скорость будет использоваться. После этого остаётся нажать клавишу старта, и программа начнёт автоматически играть мелодию. Если нужно остановить воспроизведение, достаточно нажать клавишу стопа.
# Ограничения и особенности
Скрипт сильно зависит от конкретной раскладки виртуального пианино, поэтому английская раскладка обязательна. Если окно игры неактивно, клавиши могут уйти в другое приложение. Не каждый MIDI-файл одинаково хорошо подходит для такого способа воспроизведения: сложные композиции с большим количеством дорожек, очень широким диапазоном или плотными аккордами могут звучать упрощённо. В режиме melody алгоритм не гарантирует, что будет выбрана именно та мелодическая линия, которую ожидал пользователь, а лишь старается определить наиболее логичную. В режиме full слишком большая полифония может приводить к потере части нот или к неестественному звучанию. Тем не менее для большинства обычных MIDI-файлов скрипт позволяет быстро и удобно автоматизировать проигрывание музыки в виртуальном пианино.

#Запуск с выбором файла вручную | python script.py
#Запуск с указанием файла       | python script.py --file song.mid
#Играть только мелодию	         | python script.py --file song.mid --mode melody
#Играть аккорды                 | python script.py --file song.mid --mode full --max-poly 3
# Итог: этот скрипт представляет собой MIDI-плеер для виртуального пианино, который читает MIDI, адаптирует ноты под доступную клавиатуру и автоматически воспроизводит музыку через эмуляцию нажатий. Он особенно полезен для Roblox Piano и других похожих проектов, где хочется быстро проигрывать готовые композиции без ручной игры.
